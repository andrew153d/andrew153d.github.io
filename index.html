Featured Portfolio

Remote Welder Controller V1.0
The remote welder controller was made to affordably upgrade an older welder to be controlled from a distance. During the welding process, constant adjustment of the welding current is needed to adjust the heat of the weld. Brand new machines have this feature but can cost thousands of dollars more than older machines for few additional features. The motor portion is magneticallly attached to the face of the welder and controls the knob via a belt and pulley. The motor is powered from the welder via a 12V plug and powers the motor and motor driver inside of the water resistant case. The controller and motor driver communicate over ESPNow, matching the rotation of the controller knob to the motor.
Remote Welder Controller V2.0
Version 2 of the remote welder controller builds upon the success of the first revision, modifying the design to be more end user friendly, rugged, and manufacturable. Both devices are powered by on-board batteris with the cell balancing and USB-C charging. They are both water resistant and prototype boards are comforrmally coated to avoid water damage. Both devices also support OTA software update functionality, and wifi tethering so the user can control the device from their phone. This model is meant to provide much more fine control of the welder. Sensorless homing, position feedback, and a more refined user experience all improve this design over the previous. For more in-depth details on usage, features, and design, visit the github page. While this device has not been fully built in its final form, all of the physical design and most of the software are complete and functional.
Ricky Relay
Ricky Relay is a simple dektop electronics tool made to provide two relays with NC and NO positions for various test fixtures. It has been used for hundreds of hours of testing power sequences and firmware upgrade on the device under test. The python gui allows for user control, and the simple serial interface makes test automation easy with python scripts.
Cone Generator
When metal rooves are being installed, the stand pipes that need to protrude through the roof need to be covered and sealed. There is a single German company that used to sell templates for this shape. Since they have gone out of business the most common way to make this chape is to simply guess and check. To solve this problem and make a more accurate shape, I wrote a program that can take any shape of cone intersected by any roof angle and will generate a template to use to cut out the sheet metal. The template is currently being used consistely by a roofing contractor in my area.
AutoTiller
The AutoTiller was made to automatically control the tiller on a 1970 AMF Sunfish. The device uses a compass, gyroscope, and accelerometer to determine its heading and control the tiller based on desired heading and current heading. The device gets mounted on the tiller with a spool of kevlar line attaching it to each side of the sailboat. The function button allows live adjustment of the PID loop, and allows accurate control of the sailboat.
KNN Hardware Classifier
The KNN hardware classifier uses a webcam to take images of a piece of hardware and classify it as a specific type of hardware. The classifier is trained on a dataset of 1000 images of 5 different types of hardware. It avoids the use of common frameworks like OpenCV because its final form would be tested on a bare-metal microcontroller. It achieves ~95% successful classification rate by using a custom implementation of the KNN algorithm optimized for the microcontroller environment. This project demonstrates the feasibility of running machine learning algorithms on constrained hardware.
Smart Pull
The Smart Pull is a WiFi-connected device designed to control other devices on the network using MQTT. It operates like a chain pull switch, but smarter. Using Home Assistant, the Smart Pull can control lights, with a quick pull turning the light on and off, while slowly pulling make the smart pull operate like a slider, dimming the light.
Cyber Deck
The CyberDeck project houses a Raspberry Pi 5, 5G modem, and BladeRF xA4 in a small pelican case. The platform allows for full control of the rf spectrum and remote connectivity to the cloud. The device can be powered with a usbc battery bank, allowing for complete portability. The gateway uses several open source SDR projects as well as utilities I have written. Its main use right now is as a radio logger. The device records IQ samples, demodulates them at several frequencies, and logs any transmissions present.
Reliable UDP
This project implements a reliable udp protocol in bot C and C#, allowing it to transfer data beween Windows machines and devices running C. The C application is single threaded to allow for porting to bare metal microcontrollers. The protocol features packer re-ordering, tamper detection, and multiple packet retransmission.
